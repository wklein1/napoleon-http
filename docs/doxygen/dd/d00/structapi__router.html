Detailed Description

Mutable registry backed by caller-provided storage (fixed capacity).

Note
Ownership/lifetime:
  • routes points to storage owned by the caller; capacity is max_routes.
  • prefix and each route path are not copied; they must remain valid.
  • Not thread-safe for concurrent registration/handling.
