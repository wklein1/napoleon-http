Detailed Description

Small filesystem abstraction layer used by the static file router.

The API separates a generic virtual FS handle (fs) from specific implementations (provided via vtables in fs_ops and fs_file_ops). Each concrete implementation stores its own state in fs::ctx and may embed fs_file as the first field of its concrete file handle to support polymorphic calls.

Convention in this codebase: callers pass paths relative to fs::root.

◆ fs_ensure_dir()

int fs_ensure_dir ( struct fs * vfs,
const char * path,
bool recursive )

Ensure that path exists and is a directory; create it if necessary.

Convenience wrapper that first checks the node at path:

  • If it exists and is a directory, return FS_OK.
  • If it exists and is not a directory, return FS_ERROR.
  • If it does not exist, call fs_mkdir with the given recursive flag.

This function resolves path under fs::root and never escapes the root.

Parameters
vfsFilesystem handle (must not be NULL).
pathDirectory path relative to fs::root (leading '/' allowed).
recursiveIf true, create missing parents (mkdir -p semantics) when absent.
Returns
FS_OK on success; FS_INVALID on bad arguments or rejected traversal; FS_NOT_FOUND if parents are missing and recursive is false; FS_NOT_SUPPORTED if the concrete filesystem does not support directory creation; FS_ERROR on other failures (e.g., permission denied, type conflict).
Note
Pass an empty string or "/" to ensure the VFS root exists.

◆ fs_mkdir()

int fs_mkdir ( struct fs * vfs,
const char * path,
bool recursive )

Create a directory relative to the filesystem root.

Resolves path under fs::root and attempts to create the directory. When recursive is true, missing parent components are created as well (i.e., “mkdir -p” semantics); existing components are treated as success.

Semantics:

  • If the target directory already exists, return FS_OK.
  • If a parent is missing and recursive is false, return FS_NOT_FOUND.
  • If a non-directory already exists at path, return FS_ERROR.
Parameters
vfsFilesystem handle (must not be NULL).
pathDirectory path relative to fs::root (leading '/' allowed).
recursiveIf true, create missing parents; if false, create only the leaf.
Returns
FS_OK on success; FS_INVALID on bad arguments or rejected traversal; FS_NOT_FOUND if parents are missing and recursive is false; FS_NOT_SUPPORTED if the concrete filesystem cannot create directories; FS_ERROR on other failures (e.g., permission denied, type conflicts).

◆ fs_read_all()

ssize_t fs_read_all ( struct fs_file * file,
void * buffer,
size_t cap )

Attempt to read exactly cap bytes unless EOF or an error occurs.

Reads repeatedly until either cap bytes have been read, EOF is reached, or an error occurs. If EOF occurs before cap bytes are read, the function returns the number of bytes actually read (which may be less than cap).

Semantics:

  • If cap == 0, the call is a no-op and returns 0.
  • On success, returns the total number of bytes read in [0, cap].
  • On error, returns a negative fs_return_codes value; any bytes read before the error remain in buffer.
Parameters
fileOpen file handle (must not be NULL).
bufferDestination buffer (must not be NULL when cap > 0).
capTotal number of bytes to read.
Returns
Bytes read (>= 0), 0 on EOF, or < 0 on error.

Read at most cap bytes from an open file into buffer.

This function may return fewer than cap bytes even if more data is available (i.e., a short read). A return value of 0 indicates end-of-file.

Semantics:

  • If cap == 0, the call is a no-op and returns 0.
  • On success, returns the number of bytes read (>= 0).
  • On error, returns a negative fs_return_codes value.

bufferDestination buffer (must not be NULL when cap > 0).
capMaximum number of bytes to read.
Returns
Bytes read (>= 0), 0 on EOF, or < 0 on error.
