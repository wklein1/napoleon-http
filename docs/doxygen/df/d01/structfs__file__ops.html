Detailed Description

Per-open-file operations (vtable).

Implementations provide these Functions to support reading, optional seeking, and closing a concrete file handle. The first field of the concrete handle should be an embedded fs_file so callers can dispatch via fs_file::ops.

◆ read_all

ssize_t(* fs_file_ops::read_all) (struct fs_file *file, void *buffer, size_t cap)

Implementation for “read until cap bytes or EOF/error”.

Should attempt to fill buffer with exactly cap bytes unless EOF or an error occurs. On EOF before cap, return the number of bytes actually read (>= 0). On error, return a negative fs_return_codes.

◆ read_some

ssize_t(* fs_file_ops::read_some) (struct fs_file *file, void *buffer, size_t cap)

Implementation for “read up to cap bytes”.

Should perform a best-effort read and may return fewer than cap bytes (short read). Return 0 on EOF or a negative fs_return_codes value on error.
