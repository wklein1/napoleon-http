Detailed Description

Describes a response to be serialized.

Note
Ownership/lifetime:
  • content_type must point to a NUL-terminated string valid until serialization completes; it may be NULL to to use the implementationâ€™s default policy (e.g., omit or choose a safe default).
  • extra_headers may be NULL or point to an array of http_header of length extra_headers_count; each name/value must be NUL-terminated and must not contain CR/LF.
  • body may be NULL or point to a buffer of length content_length.
  • http_response_clear() will free extra_headers and body when the corresponding owned flags are set. The struct itself is never freed by http_response_clear().
