Detailed Description

Filesystem root operations (vtable).

Implementations MUST provide all functions in this table. If a concrete filesystem cannot perform a given operation, it MUST still supply a stub that returns an appropriate error (e.g., FS_NOT_SUPPORTED).

Semantics & conventions:

  • All path arguments are interpreted relative to fs::root. A leading '/' MAY be accepted and treated as relative to the root.
  • Backends MUST prevent path traversal outside the root (reject "..").
  • Return values follow fs_return_codes (non-negative success, negative error).
  • Thread-safety is implementation-defined; document if additional external synchronization is required.

int(* fs_ops::mkdir) (struct fs *vfs, const char *path, bool recursive)

Create a directory at path (optionally with parents).

If recursive is true, create missing parent components as well (i.e., “mkdir -p” semantics). Existing components (EEXIST) should be treated as success. Implementations should honor the process umask and may retry on EINTR.

Parameters
vfsFilesystem handle (must not be NULL).
pathDirectory path relative to fs::root (leading '/' allowed).
recursiveIf true, create missing parents; if false, create only the leaf.
Returns
FS_OK on success (including when the directory already exists); FS_NOT_FOUND if parents are missing and recursive is false; FS_INVALID on bad arguments or rejected traversal; FS_NOT_SUPPORTED if the filesystem does not support directory creation; FS_ERROR on other failures.
