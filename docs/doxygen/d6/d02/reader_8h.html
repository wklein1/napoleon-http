Detailed Description

Utility functions for safe and reliable reading from file descriptors.

These helpers wrap the POSIX read(2) system call to simplify common I/O patterns, including handling of EINTR and ensuring complete reads for a fixed-length buffer.

ssize_t read_all ( int fd,

Attempt to read exactly len bytes from a file descriptor.

Internally calls read_some in a loop until either the requested number of bytes has been read, EOF is reached, or an error occurs.

If EOF occurs before len bytes could be read, the function returns the number of bytes actually read (which may be less than len).

Parameters
fdFile descriptor to read from (e.g., a socket or file).
bufferDestination buffer to store the data (must be at least len bytes).
lenNumber of bytes to read in total.
Returns
  • Number of bytes actually read (may be less than len if EOF reached).
  • -1 on error (with errno set).

Attempt to read up to len bytes from a file descriptor.

This is a thin wrapper around read(2) that transparently retries if the call is interrupted by a signal (errno == EINTR).

Unlike read_all, this function may return fewer than len bytes even if more data is available, depending on the underlying stream and buffering.
